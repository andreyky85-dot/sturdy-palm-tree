# Пункт 2: TikTok и ручной транскрипт — подробная инструкция

Речь идёт о том, чтобы пользователь мог генерировать посты не только из YouTube-ссылки, но и из **вставленного вручную текста транскрипта**. Так можно обрабатывать TikTok, подкасты, свои заметки или любой источник, откуда вы просто копируете текст.

---

## Зачем это нужно

| Сейчас | После пункта 2 |
|--------|-----------------|
| Работает только YouTube: вставил ссылку → мы тянем транскрипт с YouTube. | YouTube по-прежнему по ссылке; **плюс** можно вставить любой текст (TikTok, подкаст, свой текст) и получить те же 10 Twitter, 5 LinkedIn, 3 TikTok-идеи и 1 blog summary. |

У TikTok нет публичного API для транскрипта. Пользователь может:
- включить субтитры в приложении и переписать/скопировать текст;
- использовать сторонний сервис для расшифровки и скопировать результат;
- записать речь от руки.

Мы не тянем транскрипт с TikTok автоматически — мы даём поле «Вставить транскрипт» и генерируем посты из этого текста.

---

## Что именно делается

1. **API `/api/generate`**  
   Принимает **либо** `videoUrl` (как сейчас), **либо** `transcript` (строка с текстом).  
   - Если передан `transcript` и он не пустой — используем его и не вызываем YouTube.  
   - Если передан только `videoUrl` — работа как раньше: получаем транскрипт через `fetchTranscript(videoUrl)`.

2. **Страница Generator**  
   Два способа ввода (переключатель или табы):
   - **«YouTube URL»** — поле для ссылки (как сейчас).
   - **«Вставить транскрипт»** — текстовое поле (textarea) для вставки текста (TikTok, подкаст, что угодно).

   При отправке формы:
   - если выбран URL и поле URL не пустое — отправляем `{ videoUrl }`;
   - если выбран транскрипт и поле не пустое — отправляем `{ transcript }`.

3. **Валидация**  
   - Минимальная длина транскрипта (например 50–100 символов), чтобы не гонять в OpenAI совсем короткий текст.
   - Ограничение сверху (например 30 000–50 000 символов), как в текущем ограничении для промпта.

4. **Тексты в интерфейсе**  
   - Подпись вроде: «Или вставьте транскрипт вручную (TikTok, подкаст, любой текст)».
   - В FAQ можно добавить пункт: «Как использовать TikTok?» — ответ: включите субтитры или скопируйте текст из другого сервиса и вставьте в поле «Вставить транскрипт».

---

## Логика на бэкенде (псевдокод)

```
POST /api/generate
body: { videoUrl?: string, transcript?: string }

1. Если есть transcript и он не пустой:
   - проверить длину (мин. 50, макс. 30000 символов);
   - использовать transcript как текст для OpenAI.
2. Иначе если есть videoUrl:
   - как сейчас: transcript = await fetchTranscript(videoUrl).
3. Иначе:
   - 400: "Provide videoUrl or transcript".
4. Дальше общее: result = await generateContentFromTranscript(transcript); incrementUsage; return result.
```

Реализация уже в коде: схема принимает `videoUrl` и `transcript` (опционально), выбор источника в `route.ts`, на генераторе — переключатель и textarea.

## Где взять транскрипт для TikTok

Пользователям можно подсказать в FAQ или под полем:

- Включить субтитры в TikTok и переписать/скопировать текст (если доступно).
- Использовать любой сторонний сервис распознавания речи по аудио/видео (например расшифровка с загрузкой файла), затем скопировать результат в «Вставить транскрипт».

Мы не интегрируем такие сервисы в продукт — только даём возможность вставить уже готовый текст.
