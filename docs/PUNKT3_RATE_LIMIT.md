# Пункт 3: Rate limit по IP для /api/generate

Ограничение частоты запросов к API по IP защищает от злоупотреблений и DDoS: один и тот же IP не может бесконечно дергать генерацию без авторизации или поверх лимита пользователя.

---

## Зачем это нужно

| Без rate limit по IP | С rate limit по IP |
|---------------------|---------------------|
| Кто-то может слать много запросов с одного IP (скрипт, бот), расходуя ваш лимит OpenAI и нагружая сервер. | С каждого IP допускается не более N запросов в минуту (или в окне). Лишние получают 429. |

Лимит **по пользователю** (Free 5/мес, Pro без лимита) уже есть. Rate limit **по IP** дополняет: ограничивает запросы до входа, с одного IP при множестве аккаунтов, и просто «спам» запросами.

---

## Как реализовано

1. **Стек:** [Upstash Redis](https://upstash.com) + [@upstash/ratelimit](https://github.com/upstash/ratelimit).  
   Upstash даёт бесплатный tier Redis по REST; подходит для serverless (Vercel).

2. **Переменные окружения:**
   - `UPSTASH_REDIS_REST_URL` — REST URL вашего Upstash Redis.
   - `UPSTASH_REDIS_REST_TOKEN` — REST token.

   Если переменных нет (например, локально без Redis), rate limit **не применяется**, чтобы не ломать разработку.

3. **Правило:** например **10 запросов в 1 минуту на IP** для `POST /api/generate`.  
   Идентификатор — IP запроса (`x-forwarded-for` или `x-real-ip` на Vercel, иначе `request.headers.get("x-real-ip")` и т.п.).

4. **Порядок в обработчике:** после проверки авторизации и лимита по пользователю — проверка rate limit по IP; при превышении — ответ `429 Too Many Requests` с текстом вроде «Too many requests. Try again in X seconds.» и заголовком `Retry-After` при возможности.

5. **Лимит по пользователю остаётся главным:** сначала проверяется сессия и `checkUsage(userId)`; затем, если запрос допустим по пользователю, проверяется rate limit по IP. Так один залогиненный пользователь не «съедает» весь лимит по IP для офиса.

---

## Настройка Upstash (кратко)

1. Зайти на [console.upstash.com](https://console.upstash.com), создать Redis database (регион ближе к Vercel).
2. В деталях БД скопировать **REST URL** и **REST TOKEN**.
3. В Vercel (или в `.env.local`) задать:
   - `UPSTASH_REDIS_REST_URL=...`
   - `UPSTASH_REDIS_REST_TOKEN=...`

Либо использовать **Vercel KV**: в проекте Vercel → Storage → Create KV Database (на базе Upstash); переменные подставятся автоматически (часто те же имена или с префиксом `UPSTASH_`).

---

## Параметры (при желании поменять)

В коде заданы константы:
- **Окно:** 60 секунд.
- **Максимум запросов в окне:** 10 на IP.

Имеет смысл не ставить слишком маленькое значение (мешают легитимным пользователям с общим IP), и не слишком большое (чтобы бот не мог бесконечно слать запросы). Для старта 10/мин — разумный компромисс.
