# Пункт 4: Тесты (unit + e2e)

Unit-тесты проверяют логику без реальных API и БД (моки). E2E-тесты проверяют сценарии в браузере (лендинг, генератор, при необходимости логин).

---

## Зачем это нужно

| Тип | Что даёт |
|-----|----------|
| **Unit** | Уверенность в логике `checkUsage` / парсинга ответа OpenAI; рефакторинг не ломает поведение. |
| **E2E** | Проверка, что ключевые страницы открываются, форма и кнопки работают, после деплоя ничего не разъехалось. |

---

## Что реализовано

### 1. Unit-тесты (Vitest)

- **Запуск:** `npm run test` (или `npx vitest run`).
- **Конфиг:** `vitest.config.ts` (alias `@/` → корень проекта).
- **Файлы:**
  - `lib/usage.test.ts` — тесты для `checkUsage` и `incrementUsage` с моком Prisma:
    - пользователь не найден → `allowed: false`, `used: 0`, `limit: 0`;
    - Free-план, использовано меньше лимита → `allowed: true`;
    - Free-план, использовано >= 5 → `allowed: false`;
    - Pro-план → всегда `allowed: true`, `limit: 0`;
    - `incrementUsage` вызывает `prisma.generation.create`.
  - При желании можно добавить тесты парсинга ответа OpenAI (отдельная функция или мок ответа).

### 2. E2E-тесты (Playwright)

- **Запуск:** один раз выполнить `npx playwright install` (установка браузеров), затем `npm run test:e2e`. Локально при отсутствии CI Playwright сам поднимет `npm run dev` и дождётся готовности.
- **Конфиг:** `playwright.config.ts` — baseURL localhost:3000, один worker по умолчанию.
- **Сценарии:**
  - **Лендинг:** главная открывается, есть заголовок/hero, секции Features, Pricing, FAQ, футер.
  - **Генератор:** страница `/generator` открывается; отображается либо блок «Sign in to generate», либо форма (поле URL/транскрипт и кнопка Generate).
  - Логин через Google в E2E не автоматизирован (нужен тестовый аккаунт и стабильные селекторы); при необходимости можно добавить мок NextAuth для e2e.

### 3. Скрипты в package.json

- `"test": "vitest run"`
- `"test:watch": "vitest"`
- `"test:e2e": "playwright test"`

### 4. CI (опционально)

В GitHub Actions можно запускать:
- `npm run test` — на каждый PR;
- `npm run test:e2e` — при деплое или по расписанию (нужен запущенный `npm run dev` или деплой-preview).

---

## Запуск локально

```bash
# Unit
npm install
npm run test

# E2E (один раз установить браузеры)
npx playwright install
npm run dev   # в одном терминале
npm run test:e2e   # в другом
```

Для e2e в CI обычно поднимают сервер в job и ждут готовности, затем запускают `playwright test`.
